# Shared Components

The skill creates three shared React components in addition to `JsonLd.tsx` (covered in `nextjs-patterns/jsonld-script-component.md`). These components encapsulate the recurring AEO blocks every optimized page uses, keeping page files clean and changes centralized.

This file specifies each component's API, source code, and usage rules.

The three components are:

1. **`MicroAnswer.tsx`** — the 40 to 80 word answer block under H1 on every page.
2. **`ExpertQuote.tsx`** — the doctor-attributed clinical insight block.
3. **`FAQBlock.tsx`** — the entity-specific FAQ section with embedded FAQPage schema.

---

## Component creation rules

The skill creates each component once per project, in `components/`. If a compatible component already exists with a different name (e.g., `Hero.tsx`, `DoctorQuote.tsx`, `FaqList.tsx`), the skill detects it and:

- If API is compatible: uses the existing component, logs the reuse.
- If API is incompatible: creates the skill's version under the canonical name and logs:

```
[COMPONENT CREATED] components/MicroAnswer.tsx created.
                    Project already had components/Hero.tsx but its API
                    does not match. Both components now exist.
                    Optimized pages use the new MicroAnswer component.
```

The skill never deletes existing components.

## 1. MicroAnswer.tsx

### Purpose

Renders the 40 to 80 word direct answer that sits immediately under the H1 on every page. This is the single most important block for AI extraction. AI systems lift this block verbatim into citations.

### Props

```tsx
type MicroAnswerProps = {
  practiceName: string;       // e.g., "Bright Smiles Dental"
  city: string;               // e.g., "Calgary"
  service?: string;           // e.g., "dental implants" — required for service pages
  doctorName?: string;        // e.g., "Dr. Sarah Chen, DDS, MAGD, FAAID" — required when relevant
  quantitative: string;       // e.g., "2,400+ placed since 2010"
  differentiator: string;     // e.g., "AAID Fellow on staff, same-day consultations"
  emergencyHook?: boolean;    // adds "Same-day appointments available" line
};
```

### Behavior

- Renders a styled block (visually distinct from regular paragraph text — typically a left border or shaded background).
- Renders 40 to 80 words. The skill enforces this at content generation time, not at component time.
- All four entities (practice, city, doctor, service) appear when relevant to the page type.
- Uses semantic HTML: `<aside>` or `<section>` with `aria-label="Quick answer"`.

### Source code

```tsx
type MicroAnswerProps = {
  practiceName: string;
  city: string;
  service?: string;
  doctorName?: string;
  quantitative: string;
  differentiator: string;
  emergencyHook?: boolean;
};

/**
 * Renders the AEO Micro-Answer block: a 40-80 word direct answer immediately
 * under the H1 on every page. This block is what AI systems extract for
 * citations.
 *
 * Generated by the pracpros-dental-aeo skill.
 */
export function MicroAnswer({
  practiceName,
  city,
  service,
  doctorName,
  quantitative,
  differentiator,
  emergencyHook,
}: MicroAnswerProps) {
  return (
    <aside
      aria-label="Quick answer"
      className="border-l-4 border-primary bg-muted/40 py-4 pl-5 pr-4 my-6 rounded-r-md"
    >
      <p className="text-base leading-relaxed">
        <strong>{practiceName}</strong>
        {service && <> provides <strong>{service}</strong></>}
        <> in {city}</>
        {doctorName && <>, performed by {doctorName}</>}
        . With <strong>{quantitative}</strong>, the practice is differentiated by{" "}
        {differentiator}.
        {emergencyHook && (
          <> Same-day appointments are available.</>
        )}
      </p>
    </aside>
  );
}
```

The actual sentence structure varies — the skill generates the exact prose at content time and may pass it as a `children` prop instead of composing from atoms, depending on phrasing needs. The skill chooses based on whether natural prose can fit into the prop slots cleanly.

### Alternative API: `children`-based

For pages where the prose doesn't fit the slotted template above, the skill uses this simpler API:

```tsx
type MicroAnswerProps = {
  children: React.ReactNode;
};

export function MicroAnswer({ children }: MicroAnswerProps) {
  return (
    <aside
      aria-label="Quick answer"
      className="border-l-4 border-primary bg-muted/40 py-4 pl-5 pr-4 my-6 rounded-r-md"
    >
      <p className="text-base leading-relaxed">{children}</p>
    </aside>
  );
}
```

Usage:

```tsx
<MicroAnswer>
  <strong>Bright Smiles Dental</strong> in Calgary has placed{" "}
  <strong>over 2,400 dental implants since 2010</strong>, with a 98.7% success rate
  documented by Dr. Sarah Chen (AAID Fellow). Most candidates qualify for
  same-day implant placement after a CBCT-supported consultation.
</MicroAnswer>
```

The skill defaults to the children-based API for v1.0.0 because content phrasing varies too much for clean prop slotting. The slotted version is documented as an alternative for projects that prefer it.

## 2. ExpertQuote.tsx

### Purpose

Renders a doctor-attributed clinical insight block. This is the human voice that elevates content from "marketing copy" to "expertise". Required at least once per service page, doctor page, and most blog posts.

### Props

```tsx
type ExpertQuoteProps = {
  doctorName: string;          // e.g., "Dr. Sarah Chen"
  credentials: string;         // e.g., "DDS, MAGD, FAAID"
  quote: string;               // 30 to 80 words of clinical insight
  doctorPhotoUrl?: string;     // optional headshot
  doctorProfileUrl?: string;   // optional link to doctor page
};
```

### Behavior

- Visually distinct block (typically a card with a quote icon and the doctor's name + credentials prominently displayed).
- Quote text is the doctor's first-person voice ("In my practice...", "What I see most often...").
- Optionally renders a small headshot if `doctorPhotoUrl` is provided.
- Optionally links to the doctor's full profile page.
- Uses semantic `<blockquote>` with `<cite>` for accessibility.

### Source code

```tsx
import Image from "next/image";
import Link from "next/link";

type ExpertQuoteProps = {
  doctorName: string;
  credentials: string;
  quote: string;
  doctorPhotoUrl?: string;
  doctorProfileUrl?: string;
};

/**
 * Renders a clinical insight quote attributed to a specific doctor.
 *
 * Generated by the pracpros-dental-aeo skill.
 */
export function ExpertQuote({
  doctorName,
  credentials,
  quote,
  doctorPhotoUrl,
  doctorProfileUrl,
}: ExpertQuoteProps) {
  const nameWithCreds = `${doctorName}, ${credentials}`;

  return (
    <figure className="my-8 rounded-lg border border-border bg-card p-6">
      <blockquote className="text-lg italic leading-relaxed text-foreground">
        <p>"{quote}"</p>
      </blockquote>
      <figcaption className="mt-4 flex items-center gap-3">
        {doctorPhotoUrl && (
          <Image
            src={doctorPhotoUrl}
            alt={`${nameWithCreds} headshot`}
            width={56}
            height={56}
            className="rounded-full object-cover"
          />
        )}
        <cite className="not-italic">
          <span className="font-semibold">
            {doctorProfileUrl ? (
              <Link href={doctorProfileUrl} className="hover:underline">
                {nameWithCreds}
              </Link>
            ) : (
              nameWithCreds
            )}
          </span>
        </cite>
      </figcaption>
    </figure>
  );
}
```

### Quote content rules

The quote string passed to `ExpertQuote` must:

- Be in the doctor's first-person voice.
- Contain at least one specific clinical observation.
- Reference real practice data when available (procedures placed, years of experience).
- Avoid marketing language ("amazing results", "world-class care").
- Be 30 to 80 words.

The skill generates the quote text from `doctors[].procedures_completed`, `doctors[].years_in_practice`, and `services[].technology_used` data in the brief.

Example acceptable quote:

```
"In my practice, the most reliable predictor of long-term implant success is
adequate bone volume confirmed on CBCT. Of the 2,400+ implants I've placed
since 2010, the small number that have failed share a common factor: starting
volume that should have warranted grafting first. We've made CBCT-supported
candidacy our default screening step."
```

Example unacceptable quote (marketing fluff):

```
"At Bright Smiles Dental, we're passionate about delivering world-class
implant care to every patient we see. Our team is committed to excellence
and your satisfaction is our top priority."
```

The skill never generates the second style.

## 3. FAQBlock.tsx

### Purpose

Renders 4 to 8 entity-specific FAQs at the bottom of money pages, blog posts, and other pages where structured FAQs add value. The component also generates and emits the corresponding FAQPage JSON-LD.

### Props

```tsx
type FAQItem = {
  question: string;
  answer: string;
};

type FAQBlockProps = {
  pageUrl: string;             // for FAQPage @id (e.g., "https://brightsmilesdental.ca/dental-implants")
  faqs: FAQItem[];             // 4 to 8 items
  heading?: string;            // optional override; defaults to "Frequently Asked Questions"
};
```

### Behavior

- Renders an H2 heading ("Frequently Asked Questions" or custom).
- Renders each FAQ as a `<details>` element for native expand/collapse, or as a styled accordion if the project has shadcn/ui or Radix primitives available.
- Emits a `<script type="application/ld+json">` with FAQPage schema embedded in the same component.
- The schema script uses the `<JsonLd>` component internally to keep DOM consistent.
- Uses semantic HTML: `<section>` containing `<details>` elements with `<summary>` for questions.

### Source code

```tsx
import { JsonLd } from "./JsonLd";

type FAQItem = {
  question: string;
  answer: string;
};

type FAQBlockProps = {
  pageUrl: string;
  faqs: FAQItem[];
  heading?: string;
};

/**
 * Renders entity-specific FAQs and emits FAQPage JSON-LD schema.
 *
 * Generated by the pracpros-dental-aeo skill.
 */
export function FAQBlock({ pageUrl, faqs, heading = "Frequently Asked Questions" }: FAQBlockProps) {
  const schema = {
    "@context": "https://schema.org",
    "@type": "FAQPage",
    "@id": `${pageUrl}#faq`,
    mainEntity: faqs.map((faq) => ({
      "@type": "Question",
      name: faq.question,
      acceptedAnswer: {
        "@type": "Answer",
        text: faq.answer,
      },
    })),
  };

  return (
    <section className="my-12">
      <JsonLd data={schema} />
      <h2 className="text-2xl font-semibold mb-6">{heading}</h2>
      <div className="space-y-3">
        {faqs.map((faq, idx) => (
          <details
            key={idx}
            className="group rounded-md border border-border bg-card p-4 open:bg-muted/40"
          >
            <summary className="cursor-pointer font-medium text-base list-none flex justify-between items-center">
              <span>{faq.question}</span>
              <span className="ml-4 transition-transform group-open:rotate-180" aria-hidden="true">
                ▾
              </span>
            </summary>
            <div className="mt-3 text-base leading-relaxed text-foreground/90">
              <p>{faq.answer}</p>
            </div>
          </details>
        ))}
      </div>
    </section>
  );
}
```

### FAQ content rules per `references/01-aeo-geo-rules.md`

Every FAQ generated by the skill must:

- Be a complete question (ends in `?`).
- Contain at least 2 of: practice name, service, city.
- Have an answer 40 to 80 words long.
- Answer the question directly in the first sentence (no warm-up).

Example acceptable FAQ:

```
Q: How long do dental implants last at Bright Smiles Dental in Calgary?
A: At Bright Smiles Dental in Calgary, the 2,400+ implants Dr. Sarah Chen has
   placed since 2010 follow industry-published 95% to 98% 10-year survival
   rates when patients maintain the practice's recommended cleaning schedule.
   The titanium post itself can last decades. The crown attached to it
   typically needs replacement every 10 to 15 years.
```

Example unacceptable FAQ (generic, not entity-specific):

```
Q: How long do implants last?
A: Dental implants can last a long time with proper care.
```

The skill never generates the second style.

### Number of FAQs

| Page type | FAQ count |
|---|---|
| Service money page | 6 to 8 |
| Cost page | 4 to 6 |
| Blog post | 4 to 5 |
| Doctor page | 3 to 4 |
| Neighborhood page | 4 to 5 |
| Homepage | 3 to 5 (general practice questions) |

## Styling assumptions

The skill assumes the project uses **Tailwind CSS** because Tailwind is the dominant styling system in Next.js projects today. If `tailwind.config.{js,ts}` or `postcss.config.{js,ts}` indicates Tailwind, the components above use Tailwind classes directly.

If the project does not use Tailwind:

- The skill generates the same components without any class names.
- Adds a comment at the top of each component file: `/* Add your styling. The pracpros-dental-aeo skill generated this component without styles because Tailwind was not detected in the project. */`
- The components remain functional and semantically correct; they just look unstyled.

The skill does not add CSS modules, styled-components, or other styling frameworks. The developer is responsible for matching their existing styling system.

---

## Accessibility

All three components meet WCAG AA standards:

- `MicroAnswer` uses `<aside>` with `aria-label`.
- `ExpertQuote` uses semantic `<blockquote>` and `<cite>`.
- `FAQBlock` uses `<details>`/`<summary>` for native keyboard expand/collapse with no JavaScript required.

The skill does not add custom focus rings, custom keyboard handlers, or ARIA attributes beyond what is necessary. The native HTML semantics handle accessibility correctly.

## When the skill updates an existing component

If `components/MicroAnswer.tsx` already exists in the project, the skill checks:

1. Does it accept the same prop names? If yes, leave it alone.
2. Does it accept compatible-but-renamed props (e.g., `practice` instead of `practiceName`)? If yes, leave it alone and adjust the optimized page imports to use the existing prop names.
3. Is the existing component fundamentally different (e.g., a hero component that doesn't follow the Micro-Answer structure)? Create the skill's version under a new name like `MicroAnswerV2.tsx` and log:

```
[COMPONENT CONFLICT] components/MicroAnswer.tsx exists but does not match
                     the AEO Micro-Answer block contract.
                     Created components/MicroAnswerAEO.tsx instead.
                     Optimized pages will import MicroAnswerAEO.
                     Recommend reviewing both files and consolidating.
```

The skill prefers preservation over overwrite, even when the existing component is suboptimal. The developer reviews and consolidates.
